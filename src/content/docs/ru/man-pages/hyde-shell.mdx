## Название

`hyde-shell` - Оболочка для выполнения скриптов HyDE и разрешения окружения

## Краткий обзор

```
hyde-shell [команда] [аргументы...]
hyde-shell [--help | help | -h]
hyde-shell [-r | reload]
```

## Описание

`hyde-shell` это универсальная скрипт-оболочка, которая выполняет скрипты HyDE без расширений файлов и решает проблемы, связанные с PATH, когда пользователи могут случайно перезаписать`~/.local/lib/hyde` в своем PATH.

Расположенный в `~/.local/bin/`, `hyde-shell` автоматически обнаруживает и выполняет скрипты из каталога библиотеки HyDE, обрабатывая как скрипты Python (.py), так и shell-скрипты (.sh) без необходимости указывать расширения файлов.

## Опции

**`--help, help, -h`**
: Отображение справочного сообщения с доступными командами и информацией об использовании.

**`-r, reload`**
: Reload the HyDE environment and refresh all configurations.

## Встроенные команды

**`wallbash`**
: Выполнение скрипта wallbash для динамического управления темами и цветами.

## Выполнение скриптов

`hyde-shell` предоставляет доступ ко всем скриптам HyDE без необходимости указывать расширения файлов:

```bash
hyde-shell waybar              # Выполняет waybar.py
hyde-shell theme.import        # Выполняет theme.import.py
hyde-shell wallpaper           # Выполняет wallpaper.sh
hyde-shell screenrecord        # Выполняет screenrecord.sh
```

Оболочка автоматически обнаруживает и разрешает все скрипты в каталоге библиотеки HyDE.

## Примеры использования

Выполнение любого скрипта HyDE без расширения файла:
```bash
hyde-shell waybar --help           # Запуск waybar.py с флагом помощи
hyde-shell theme.import            # Запуск theme.import.py
hyde-shell wallpaper               # Запуск wallpaper.sh
```

Использование встроенных команд:
```bash
hyde-shell wallbash                # Выполнение скриптов wallbash в ~/.config/hyde/wallbash/scripts/

hyde-shell reload                  # Перезагрузка окружения HyDE
```

Передача аргументов базовым скриптам:
```bash
hyde-shell waybar --set layout-1   # Передача аргументов в waybar.py
```

## Ключевые особенности

- **Разрешение пути к скрипту**: Находит скрипты в `~/.local/lib/hyde/` независимо от конфигурации PATH
- **Обработка расширений**: Автоматически добавляет соответствующие расширения файлов (.py, .sh)
- **Независимость от PATH**: Работает независимо от текущей конфигурации PATH или рабочего каталога

## Файлы

**`~/.local/bin/hyde-shell`**
: Основной исполняемый скрипт-оболочка

**`~/.local/lib/hyde/`**
: Каталог, содержащий все скрипты и модули HyDE

## Примечания

- Всегда предпочитайте использовать hyde-shell вместо прямого выполнения скриптов для максимальной совместимости
- Расширения файлов необязательны и разрешаются автоматически
- Будущие версии HyDE будут все больше полагаться на hyde-shell для выполнения скриптов
